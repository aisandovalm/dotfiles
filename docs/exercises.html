Build the Muscle Memory

Ten minutes a day beats one long session. Do these roughly in order — each block assumes the one before it. The "checkpoint" drills are the real test: do them from memory, cheat sheet closed.

1 WezTerm

Warm-up

Tabs, without the mouse

  1. Open a new tab with Ctrl Shift T.
  2. Open a second new tab, this time with leader (Ctrl W, release, c).
  3. Jump straight to tab 1 with Alt 1, then tab 3 with Alt 3.
  4. Cycle forward through all tabs with Ctrl Tab, then backward with Ctrl Shift Tab.
  5. Close the extra tabs with Ctrl Shift W until one is left.
Core skill

Split and navigate a 4-pane layout

  1. Split horizontally: leader |.
  2. Split the new pane vertically: leader -.
  3. Move focus around with Ctrl Shift H/J/K/L until you can land on any of the panes without looking.
  4. Split one more time so you have 4 panes total, then close one with Ctrl Shift X.
Checkpoint — cheat sheet closed

From a single tab, in under 30 seconds:

  1. Open 2 more tabs, jump to tab 2 directly by number.
  2. Split the current pane both ways so you have 3 panes.
  3. Move focus to the top-left pane.
  4. Close one pane and one tab, confirming both.

2 herdr

Warm-up

Launch and get oriented

  1. Run herdr from any shell.
  2. Press the prefix Ctrl B then ? to see the full help overlay. Skim it once, then dismiss it.
  3. Create a new workspace: prefix then Shift N.
  4. Open the workspace picker with prefix w and switch back to your first workspace.
Core skill

Tabs, panes, and the prefix reflex

  1. Inside a workspace, open a new tab: prefix c.
  2. Split it: prefix " (horizontal), then prefix % (vertical) on the new pane.
  3. Move between the panes with prefix h j k l until it's automatic.
  4. Use goto: prefix g to fuzzy-jump straight to a specific pane by name.
  5. Close the extra tab: prefix &.
Core skill

Copy mode

  1. Run something that prints a few lines of output (e.g. ls -la).
  2. Enter copy mode: prefix y.
  3. Start a selection with v, move over a line or two, copy with y.
  4. Paste it back into the shell with Cmd V (that's WezTerm's binding, not herdr's).
Checkpoint — cheat sheet closed

From scratch:

  1. Create a new workspace and give it a label.
  2. Open a new tab, split it into 2 panes.
  3. Copy a line of text from one pane using copy mode.
  4. Detach from the session (prefix q), then reattach with herdr session attach <name> from the shell.

3 Neovim

Warm-up

The one weird trick in this config

  1. Open any file with nvim somefile.txt.
  2. Press i, type a sentence, then press Esc.
  3. Check the file on disk (in another pane) — it's already saved. That's this config's Esc → :w remap. Do this five times with different edits until reaching for Esc-to-save feels normal.
Core skill

Finding things without the mouse

  1. From inside a project folder, open nvim.
  2. Press Space f and open any file by typing part of its name.
  3. Press Space s and grep for a word you know exists somewhere in the project.
  4. Press Space b to jump between the files you've now got open as buffers.
  5. Put your cursor on a function call and press g d to jump to its definition.
Core skill

File browser and git, in-editor

  1. Press Space e to open oil.nvim. Notice it looks like a normal text buffer.
  2. Rename a file by editing its text in that buffer and saving.
  3. Back in a normal buffer, make a small edit and save it.
  4. Press Space g to open Neogit, stage the change, and write a commit.
Checkpoint — cheat sheet closed

From an empty terminal:

  1. Open Neovim in a project you know.
  2. Find a file by name, then find a specific word by grepping.
  3. Select the whole file with Ctrl a, then undo the selection.
  4. Make an edit, save it with Esc, then open Neogit and commit it.

4 Combined workflow drill

This is the one that matters — it's the actual shape of a normal working session, chaining all three tools together. Do it without the cheat sheet once you can.

Full loop

A realistic 10-minute session

  1. In WezTerm, open a new tab (Ctrl Shift T) and run herdr.
  2. In herdr, create a workspace for a project and split its pane into two: one for an editor, one for a shell.
  3. Focus the editor pane, launch nvim, and use Space f to open a file.
  4. Make a small, real edit. Save with Esc.
  5. Open Neogit (Space g), stage and commit the change.
  6. Switch to the shell pane (herdr prefix h/l) and run a command against the repo (tests, a linter, git log, anything).
  7. Copy a line of that output using herdr's copy mode (prefix y).
  8. Open a second WezTerm tab for something unrelated, then jump back to the first with Alt 1.
  9. Detach from herdr (prefix q) and reattach from the shell to prove the session survived.
Once this feels boring You've built the muscle memory. From here it's just repetition on real work — the combined drill above is a normal day, so the next rep is tomorrow's actual session.
